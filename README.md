# Web Application Vulnerability Assessment & Penetration Testing (VAPT)

## Overview

This project demonstrates a Vulnerability Assessment and Penetration Testing (VAPT) exercise performed on a vulnerable web application environment. The assessment focused on identifying security weaknesses, analyzing potential attack vectors, and documenting remediation recommendations.

The objective was to gain hands-on experience with web application security testing methodologies, reconnaissance techniques, vulnerability identification, and security reporting.

---

## Objectives

* Perform reconnaissance and information gathering.
* Identify exposed services and web resources.
* Assess web application vulnerabilities.
* Validate security weaknesses through controlled testing.
* Document findings and recommend remediation measures.

---

## Tools Used

* Kali Linux
* Nmap
* Wireshark
* Gobuster
* DVWA (Damn Vulnerable Web Application)
* Burp Suite
* Web Browser Developer Tools

---

## Activities Performed

### Reconnaissance & Enumeration

* Host discovery
* Port scanning
* Service enumeration
* Directory and file discovery

### Web Application Testing

* SQL Injection Testing
* Cross-Site Scripting (XSS) Testing
* Command Injection Testing
* Authentication and Input Validation Analysis

### Traffic Analysis

* Packet capture and inspection
* HTTP request and response analysis
* Network communication monitoring

### Reporting

* Vulnerability documentation
* Risk assessment
* Remediation recommendations
* Security best practices

---

## Key Findings

The assessment identified multiple web application vulnerabilities commonly referenced in the OWASP Top 10, including:

* SQL Injection
* Cross-Site Scripting (XSS)
* Command Injection
* Insecure Input Validation

Each finding was analyzed for potential impact and accompanied by mitigation recommendations.
